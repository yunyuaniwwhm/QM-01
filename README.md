# dev-2026-01

## 题目名称

HTML + CSS 页面基础开发：实现一个新人可完成的任务管理页面

## 这是一道什么题

请你使用 **HTML + CSS** 完成一个“任务管理页面”静态页面。

这是一道面向新人的前端基础考核题，重点考察：

- HTML 基础结构
- CSS 基础布局
- 常见页面样式实现
- 基础响应式适配
- 基本代码组织能力

本题不要求接入后端，不要求实现复杂 JavaScript 逻辑。

如果你之前没有做过类似项目，也不用紧张。这道题允许你一边学习一边完成，重点是看你能不能把一个基础页面认真做出来。

## 开发要求

请完成一个静态页面，页面至少包含以下部分。

### 1. 顶部区域
需要包含：

- 页面标题或产品名称
- 一个搜索框
- 一个按钮（例如“新建任务”）

### 2. 导航区域
可以是左侧导航，也可以是顶部下方的一排导航。

至少包含 3 个导航项，例如：

- Dashboard
- My Tasks
- Settings

要求：

- 当前选中项有明显样式
- 鼠标悬停时有样式变化

### 3. 主内容区
至少包含以下两个模块。

#### A. 数据卡片区
至少 2 张卡片，例如：

- 今日待办
- 已完成任务

#### B. 任务列表区
至少展示 4 条任务，且每条任务至少包含：

- 任务名称
- 截止日期
- 状态

状态至少区分 2 种，例如：

- 进行中
- 已完成

### 4. 附加信息模块
下面任选 1 个实现即可：

- 今日日程
- 最近活动
- 团队成员
- 温馨提示

## 布局要求

### 桌面端
- 页面结构清晰
- 各模块之间有明显分区
- 内容排版整齐

### 移动端
当屏幕宽度较小时，页面需要：

- 内容仍然可以正常阅读
- 卡片和列表能合理换行或上下排列
- 不出现明显重叠、溢出、错位

---

## 技术要求

### 必须满足

- 使用原生 HTML + CSS 完成
- 至少使用 1 次 `flex` 或 `grid`
- 使用基本合理的标签，例如：
  - `header`
  - `main`
  - `section`
  - `nav`
- 至少有基础交互状态：
  - `:hover`
  - `:focus`

### 不建议做法

以下做法会影响评分：

- 大量使用 `position: absolute` 硬拼布局
- 整个页面几乎全靠随意嵌套的 `div`
- 直接套用完整后台模板
- 不做移动端适配
