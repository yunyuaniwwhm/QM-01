# 任务管理页面

## 项目简介

这是一个使用 HTML + CSS 完成的任务管理静态页面，是“启梦团队”2026开发方向的第一道考核题目。页面实现了清晰的结构、完整的样式和基础的响应式布局。

## 页面包含的内容

- **顶部区域**：包含产品标题、搜索框和“新建任务”按钮。
- **导航区域**：侧边栏导航，包含“工作台”、“我的任务”、“日历”、“设置”四个选项，当前选中项有高亮效果。
- **主内容区**：
  - **数据卡片区**：包含“今日待办”、“已完成”、“进行中”三张数据卡片。
  - **任务列表区**：展示了四条任务，每条包含任务名称、描述、截止日期、负责人等信息，并区分“进行中”和“已完成”状态。
- **附加信息模块**：展示了“今日日程”。

## 我是怎么实现的

### HTML 结构

我把页面分成了以下几个主要部分，并使用了语义化标签：

- `<header>`：用于包裹整个顶部区域。
- `<nav>`：用于包裹侧边导航栏。
- `<main>`：用于包裹所有核心内容。
- `<section>`：用于将内容分为“数据概览”、“任务列表”、“今日日程”三个独立的模块。
- `<footer>`：用于放置页脚信息。

### CSS 布局

我主要使用了 **Flexbox** 来实现布局：

- **顶部区域 (`.site-header`)**：使用 `display: flex;` 让标题、搜索框和按钮水平排列并均匀分布。
- **主体容器 (`.app-container`)**：使用 `display: flex;` 实现导航栏和主内容区的左右布局。
- **数据卡片区 (`.stats-cards`)**：使用 `display: flex;` 并配合 `flex-wrap: wrap;` 让卡片水平排列，并在空间不足时自动换行。
- **任务列表项 (`.task-item`)**：使用 `display: flex;` 和 `justify-content: space-between;` 让任务详情和操作按钮分居左右两端。

**交互状态**：为按钮、导航链接、输入框等元素添加了 `:hover` 和 `:focus` 样式，以提供视觉反馈。

### 响应式处理

当屏幕宽度小于 **768px**（移动端）时，我做了以下调整：

- **顶部区域**：改为垂直堆叠布局，按钮和搜索框宽度变为100%。
- **导航区域**：从左侧侧边栏变为顶部的水平导航条。
- **数据卡片**：从横向排列变为纵向堆叠，每张卡片占满宽度。
- **任务列表项**：从左右布局变为上下布局，操作按钮移动到任务详情下方。
