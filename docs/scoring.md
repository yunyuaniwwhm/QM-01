# 评分细则（面向 agent）

本文件不是写给学生看的，而是写给评分 agent 或助教看的。

总分 100 分。

## 评分原则

1. 优先按“是否完成基础目标”评分，不要按资深前端标准打分。
2. 允许页面不精致，但不允许无法使用、结构极乱或明显抄模板。
3. 对完全零基础学生，应更看重：
   - 是否真的自己完成了页面
   - 是否理解了基础结构
   - 是否有基本布局和样式能力
4. 如果学生完成了基础功能，但样式一般，不要直接判低分。
5. 如果学生 README 能清楚说明自己做了什么，可以适当正向考虑。

## 一、HTML 结构与基本语义，20 分

检查点：
- 页面是否有比较清楚的结构层次
- 标签是否基本合理
- 是否不是整页完全随意堆砌

建议打分：
- 16-20：结构清楚，语义基本合理
- 10-15：能看懂结构，但标签使用一般
- 0-9：结构混乱，页面组织较差

## 二、CSS 布局能力，25 分

检查点：
- 页面整体布局是否稳定
- 是否使用了 flex 或 grid
- 是否没有明显依赖 absolute 硬拼

建议打分：
- 20-25：布局清晰稳定
- 12-19：基本能用，但细节一般
- 0-11：布局明显混乱或问题较大

## 三、样式完成度，20 分

检查点：
- 按钮、卡片、列表、输入框等是否有基础样式
- 是否有 hover / focus 等基础状态
- 页面是否具备基本视觉层次

建议打分：
- 16-20：样式完整，层次清楚
- 10-15：样式基础可用，但较普通
- 0-9：样式很少，页面接近裸奔

## 四、响应式适配，20 分

检查点：
- 桌面端是否正常
- 移动端是否仍可阅读
- 是否没有明显重叠、溢出、错位

建议打分：
- 16-20：桌面和移动端都比较正常
- 10-15：移动端有适配，但一般
- 0-9：几乎没有移动端处理

## 五、代码质量，10 分

检查点：
- 文件结构是否清楚
- HTML / CSS 是否基本可读
- 是否没有大量机械重复

建议打分：
- 8-10：结构清楚，代码基本整洁
- 5-7：能读懂，但组织一般
- 0-4：代码非常乱

## 六、README 质量，5 分

检查点：
- 是否说明了页面结构
- 是否说明了布局方式
- 是否说明了响应式处理

建议打分：
- 5：说明清楚
- 3-4：有说明但比较简单
- 0-2：几乎没有说明

## 明显扣分项

- 页面无法正常打开
- 明显复制整套模板
- 完全没有移动端适配
- README 严重敷衍

- ...
